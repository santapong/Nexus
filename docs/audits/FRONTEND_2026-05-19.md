# Frontend Audit — NEXUS Dashboard

**Date:** 2026-05-19
**Auditor:** Agent A1 (Frontend Audit lane)
**Scope:** `frontend/` end-to-end (read-only)
**Status:** Phase 7 complete codebase (per CLAUDE.md §2)

---

## Executive Summary

The NEXUS frontend is a small (~30 components, ~3.5k LOC), well-typed React 18 +
Vite + TanStack Query app that closely follows §17 of CLAUDE.md on the **shape** of
the code: strict TypeScript is on, there is zero `any` and zero `@ts-ignore`, server
state lives exclusively in TanStack Query hooks (the only `fetch()` call sits in
`src/api/client.ts`), and Zustand is used in exactly one store for derived UI/event
state.

However, the audit surfaced **one show-stopping defect** and several other gaps that
are unacceptable for a Phase 7 codebase that bills itself as "enterprise security by
default" (§3 principle 8):

**Top 5 findings (severity order):**

1. **CRITICAL — Auth token never sent.** `LoginPanel` stores the JWT in
   `localStorage` as `nexus_token`, but `apiFetch` (`src/api/client.ts:37-44`) never
   reads it back and never sets an `Authorization: Bearer …` header. The backend's
   `require_auth_user` (`backend/nexus/api/auth.py:147`) raises 401 on any request
   without a Bearer token. The entire authenticated surface area is effectively
   unreachable from this UI — or, in dev where auth is loose, the UI silently runs
   without workspace isolation.

2. **CRITICAL — WebSocket has zero authentication.** `agent_activity_ws`
   (`backend/nexus/api/websocket.py:15`) accepts any client and pipes
   `agent_activity:*` Redis pub/sub messages cross-workspace. The frontend connects
   to `/ws/agents` with no token (`AgentWebSocketProvider.tsx:83`). Any
   unauthenticated network peer can subscribe and observe every tenant's agent
   thinking-stream, tool calls, and task IDs in real time.

3. **HIGH — No ErrorBoundary anywhere.** `grep` for `ErrorBoundary`,
   `componentDidCatch`, `getDerivedStateFromError` returns zero hits. A single
   uncaught render error in any page will white-screen the entire dashboard with no
   recovery path.

4. **HIGH — Irreversible actions ship without confirmation.** Per §20 NEVER rule 1,
   irreversible actions require explicit human confirmation. The UI has none of the
   "confirm-with-typing" friction the task spec calls out for high-risk approvals:
   `ApprovalPanel` (`src/components/approvals/ApprovalPanel.tsx:37-56`) approves
   `git_push` and `send_email` with a single click. `A2ATokenPanel`
   (`A2ATokenPanel.tsx:163-169`) revokes/rotates bearer tokens with one click and
   no `confirm()` dialog.

5. **HIGH — Zero code splitting; every page bundled at startup.**
   `src/router.tsx` imports every page eagerly (lines 4-13). For a SaaS dashboard
   with 11 routes and a long-lived WebSocket, initial bundle is the entire app.
   Vite build is a single chunk.

Other notable findings are tabulated below. Overall the codebase is **clean in form
but immature in security and resilience posture** — exactly the gap you'd expect
between "Phase 7 declared complete" and "ready to bill paying customers."

## Looks correct

- **TypeScript strictness:** `tsconfig.json` has `strict: true`, `noUnusedLocals`,
  `noUnusedParameters`, `noFallthroughCasesInSwitch`,
  `forceConsistentCasingInFileNames` (`tsconfig.json:14-18`). Zero `any` in `src/`.
  Zero `@ts-ignore` / `@ts-expect-error` / `@ts-nocheck`.
- **No raw `fetch()` in components.** The only `fetch(` call is inside
  `src/api/client.ts:38`. §17 MUST-2 satisfied in form (not in security — see C1).
- **No XSS sinks.** Zero `dangerouslySetInnerHTML`, zero `innerHTML`, zero `eval(`.
- **No leaked secrets in client bundle.** Only `VITE_API_URL` is referenced and it's
  used purely as a base URL. No accidental `VITE_*_SECRET` or `VITE_*_KEY`.
- **Zustand scope:** Single store (`agentEventStore`) holds derived event state
  (live agent status map, ring buffer of "thinking" entries, connection state). Not
  used for server data — server data lives in TanStack Query. §17 MUST-4 satisfied.
- **API client URL handling.** `VITE_API_URL` env-driven, sensible default.
- **Hooks separation.** All API calls funnel through `src/hooks/*` and `src/api/`.
  Components do not call `apiFetch` directly. §17 MUST-3 satisfied.
- **Topic / event names not hardcoded into UI logic.** UI consumes `event` strings
  from backend without knowing Kafka topic names — clean boundary.
- **ARIA on critical buttons.** Approval Approve/Reject buttons carry sensible
  `aria-label`s (`ApprovalPanel.tsx:42,52`). Sidebar nav has `aria-expanded` /
  `aria-controls` (`Sidebar.tsx:110-111`). Live status board has section
  `aria-label`s.
- **Live-status board pulse / focus styles**: `focus-visible:ring-*` on Button cva
  variants. Good for keyboard nav.
- **Tasks list filter UX**: solid — search, status filter, sort, clear-filters
  empty state.
- **Production nginx config** has `gzip on`, `proxy_set_header`, `try_files
  $uri/index.html` — sensible SPA defaults (`Dockerfile`).

## Recommended fix order

The order below prioritizes by blast-radius and how cheap the fix is.

| Order | Item | Why first | Rough effort |
|-------|------|-----------|--------------|
| 1 | **C1: Attach JWT to every API call.** In `apiFetch`, read `localStorage.getItem('nexus_token')` and append `Authorization: Bearer …`. Add 401 interceptor that clears token and redirects to `/login`. | Without this nothing else works. | 30 min |
| 2 | **C2: Authenticate the WebSocket.** Pass the JWT as `Sec-WebSocket-Protocol` subprotocol (preferred — not logged by proxies) or as a query string with a short-lived ticket. Backend must validate, then `psubscribe` to `agent_activity:{workspace_id}:*` instead of `agent_activity:*`. | Plugs a cross-tenant data leak. Requires backend change too. | 2 h |
| 3 | **H7: Add auth gate.** A `RequireAuth` component reading the same token; redirect to `/login` if missing. Pair with item 1's 401 interceptor. | Closes the "anonymous app" UX hole. | 1 h |
| 4 | **C3 / M3: Move JWT off localStorage.** Either an `httpOnly` cookie (requires CSRF protection on state-changing endpoints — backend currently has none visible) or keep in memory + silent refresh. Add a CSP meta tag in `index.html`. | Defense in depth against XSS token theft. | 4 h |
| 5 | **H2 / H3: Add confirm-with-typing on irreversible actions.** Reusable `<ConfirmDialog requireText="git_push">` component. Wire into `ApprovalPanel` (for `git_push`, `send_email`, `file_write`, `hire_external_agent`) and `A2ATokenPanel` (revoke + rotate). Also surface `agent_id` in approval rows. | Direct §20 NEVER-1 compliance gap. | 3 h |
| 6 | **H1: ErrorBoundary.** Add a top-level boundary in `App.tsx` and per-route boundaries via the `errorElement` prop in `createBrowserRouter`. | One-line failure → whole-app whitescreen today. | 1 h |
| 7 | **H5 / H6: WS exponential backoff + replay.** Replace fixed 3 s with `Math.min(30_000, 1_000 * 2 ** attempt)` + jitter. Track a `last_event_id` and send it as `?since=` on reconnect; backend replays from Redis Stream / event store. | Production reliability and consistency. | 4 h |
| 8 | **H4: Code splitting.** Convert page imports to `React.lazy()` + `<Suspense>` wrapper in router. Verify Vite emits one chunk per route. | Cheap perf win. | 1 h |
| 9 | **M1: OpenAPI-generated types.** Add `openapi-typescript` script that pulls Litestar OpenAPI JSON and regenerates `src/types/index.ts` + (optionally) `src/api/client.ts`. Wire into CI. | Eliminates whole class of drift bugs (H8). | 4 h |
| 10 | **H9 / H10 / M12: Tighten query lifecycle.** Lower poll intervals to fallback values (60 s+) when WS is open; raise default `staleTime` to 30 s; per-query overrides; flag for WS-cancelled reconnect race. | Lowers backend load by ~5-10× in idle dashboards. | 2 h |
| 11 | **M4 / M5: Accessibility pass on modal + form labels.** Pull in `@radix-ui/react-dialog` (already Shadcn-aligned) for `TaskFeedbackModal`. Add `<label htmlFor>` everywhere a `<input>`/`<textarea>` lacks one. | Phase 7 enterprise polish. | 4 h |
| 12 | **M6 / M7: Virtualize long lists + throttle WS updates.** `react-virtuoso` for `TaskListPanel`, `AuditDashboard`, replay views; `requestAnimationFrame`-based batched flush in `agentEventStore`. | Needed before any real-world high-traffic test. | 4 h |
| 13 | **M9: Surface structured backend errors.** Have `apiFetch` parse error JSON and throw a typed `ApiError`; let hooks display the real message. | UX quality. | 1 h |
| 14 | **M8 / M13 / M11 / M14: Cleanups.** Misleading badge on Sidebar; remove dead code in LiveStatusBoard; switch to `crypto.randomUUID()` for entry IDs; add output truncation toggle. | Low-effort polish. | 2 h total |

**Estimated total to close all findings:** ~35 person-hours.

**Truly blocking before any external launch:** items 1, 2, 3, 4, 5, 6 (~10 h).
