# NEXUS Project Audit — 2026-05-19

High-level audit across four areas: **Security**, **Database Performance**,
**Backend Correctness**, **Tool/MCP Integration**.

Reusable skill files for re-running each audit live in `.claude/skills/audit-*/`.

---

## Executive summary

The system is well-architected on paper (CLAUDE.md §1–11 are sound), but several
**critical safety claims in the docs are not actually enforced in code**:

| Claim in docs | Reality |
|---------------|---------|
| "Irreversible tools require human approval" (CLAUDE.md §8, §20) | `tool_file_write`, `tool_git_push`, `tool_send_email`, `tool_hire_external_agent` all execute without calling `require_approval()`. The guard is defined but never invoked. |
| "PostgreSQL Row-Level Security isolates tenants" (Phase 5 Track A) | RLS policies exist on tables, but `SET LOCAL nexus.workspace_id` is never executed on the session. Policies are dormant. |
| "Memory write must succeed before publish" (CLAUDE.md §20 rule 2) | If `write_memory()` raises, the outer `except` block publishes the result anyway. |
| "All Kafka messages HMAC-signed and verified on consume" (Phase 7) | Signing is implemented in `core/kafka/signing.py`, but `core/kafka/result_consumer.py` never validates signatures. |
| "PII sanitisation before publish" (Phase 7) | `sanitize_output()` exists in `core/sanitization.py` but is not called before episodic memory writes or Kafka publishes. |
| "OAuth tokens encrypted" (column names suggest it) | Stored as plaintext despite `_encrypted` column suffix. |

**Total: 16 Critical, 18 High, 14 Medium findings.** None of these are large
refactors — most are 1–10 line fixes. Recommended order in §5.

## 1. Security — findings

### Critical

1. **OAuth tokens stored in plaintext** (`backend/nexus/api/oauth.py:255`)
   Columns named `access_token_encrypted` / `refresh_token_encrypted` hold raw strings.
   DB read access = full account takeover of every linked Google/GitHub user.
   **Fix:** Wrap with `cryptography.fernet.Fernet` keyed off `settings.encryption_key`.

2. **RLS context never injected** (`backend/nexus/api/middleware.py:34`)
   `RLSMiddleware` stores `workspace_id` in `scope["state"]` but never calls
   `set_rls_context()` on the SQLAlchemy session. Every RLS policy is dormant.
   **Fix:** Hook Litestar's session lifecycle (`before_request` + session
   dependency) to execute `SET LOCAL nexus.workspace_id = '{uuid}'` on the
   request-scoped session before route handlers run.

3. **OAuth state parameter missing (CSRF)** (`backend/nexus/api/oauth.py:128`)
   No random `state` token generated on redirect or validated on callback.
   Attacker can link a victim's NEXUS account to the attacker's external account.
   **Fix:** Generate 32-byte token, store in Redis db:1 with 10-min TTL keyed by
   session ID, validate on callback.

4. **A2A token hashing is plain SHA-256** (`backend/nexus/integrations/a2a/auth.py:46`)
   Cacheable, precomputable. Offline dictionary attack feasible on common token
   patterns. In-memory cache adds a timing side channel.
   **Fix:** Switch to PBKDF2-HMAC-SHA256 with a per-token salt; lower cache TTL.

5. **API key endpoints bypass RLS** (`backend/nexus/api/api_keys.py:69`)
   Creates fresh session via `sqlalchemy_config.get_session()` instead of using
   the Litestar-injected session — sidesteps `RLSMiddleware` entirely.
   **Fix:** Accept `db_session: AsyncSession` as a route handler parameter.

### High

6. **Hardcoded dev signing key fallback** (`backend/nexus/core/kafka/signing.py:32`)
   `_get_signing_key()` falls back to `"nexus-dev-signing-key"` if
   `JWT_SECRET_KEY` is empty. In production that produces predictable signatures.
   **Fix:** `raise SettingsError(...)` when key is empty in non-dev environments.

7. **Result consumer skips signature validation** (`backend/nexus/core/kafka/result_consumer.py:45`)
   Accepts unsigned/forged `agent.responses` messages. An attacker who can write
   to Kafka can mark any task complete with arbitrary output.
   **Fix:** Call `validate_message_signature()` at the start of `_handle_response()`;
   route invalid messages to dead-letter.

8. **OAuth provider responses logged whole** (`backend/nexus/api/oauth.py:179`)
   `logger.error("token_exchange_failed", error=token_json)` can leak access
   tokens if the provider returns them alongside an error.
   **Fix:** Log only `error_code` / `error_description` by name.

### Medium

9. **LLM injection classifier fails open** (`backend/nexus/api/middleware.py:235`)
   On classifier exception, returns `valid=True` (task allowed through). An
   attacker who can DDoS the classifier bypasses all detection.
   **Fix:** Default to `valid=False` in production; grace allowance only in dev.

10. **Audit log has no DB-level immutability** (`backend/nexus/audit/service.py`)
    `archive_old_audit_records()` does `UPDATE audit_log SET archived_at = ...`
    — there's no CHECK constraint or trigger preventing arbitrary UPDATE.
    **Fix:** PostgreSQL trigger that rejects any UPDATE except the archival job
    (matched by setting `archived_at` only).

11. **A2A rate limiter not enforced per request** (`backend/nexus/integrations/a2a/rate_limiter.py`)
    `rate_limit_rpm` is stored on tokens but no middleware checks it per
    incoming request. Cache only.
    **Fix:** Add middleware that increments a Redis sliding-window counter on
    every authenticated A2A request and rejects above limit.

12. **No login rate limiting** (`backend/nexus/api/auth.py:121`)
    Brute force unthrottled.
    **Fix:** Per-IP + per-email Redis counter, fail at 10/min.

### Notes — looks correct

- Password hashing: PBKDF2-HMAC-SHA256, 600k iterations, `compare_digest()` ✓
- JWT expiry validated on decode ✓
- A2A token revocation + expiry ✓
- `require_approval()` flow inside `guards.py` itself is sound — the bug is that
  it's not called by the tools ✓ (see Tools §4)
- Kafka HMAC signing math is correct (`signing.py:37–73`) ✓
- No raw-SQL f-string interpolation anywhere ✓
- SOPS → KeepSave → env fallback chain ✓

## 2. Database performance — findings

### Critical (will degrade at 1k+ tasks/day)

1. **`episodic_memory.embedding` has NO ivfflat index**
   `backend/nexus/memory/episodic.py:61` runs cosine-similarity recall (`<=>`)
   on every task. Currently sequential scan.
   **Fix:** `CREATE INDEX ix_episodic_memory_embedding ON episodic_memory USING ivfflat (embedding vector_cosine_ops) WITH (lists=100);`

2. **`tasks` lacks composite index for the most common query pattern**
   `WHERE workspace_id = ? AND status = ? ORDER BY created_at DESC` is the
   dashboard list query. Current indexes: `(workspace_id, created_at)` and
   `(status)` separately — planner can only use one.
   **Fix:** `CREATE INDEX ix_tasks_workspace_status_created ON tasks (workspace_id, status, created_at DESC);`

3. **`llm_usage` lacks `(model_name, created_at)` index**
   Analytics queries (`analytics.py:316`, `:625`) group by model with a date
   filter. At 10k rows/day, this becomes a full-table scan within weeks.
   **Fix:** `CREATE INDEX ix_llm_usage_model_created ON llm_usage (model_name, created_at DESC);`

### High

4. **`semantic_memory(agent_id, namespace)` not indexed**
   Hot path — every task loads semantic facts. Currently filtered by `agent_id`
   then linear scan of namespaces.
   **Fix:** `CREATE INDEX ix_semantic_memory_agent_namespace ON semantic_memory (agent_id, namespace);`

5. **`audit_log(event_type, created_at)` not indexed**
   `audit.py:72` filters by `event_type`. Append-only table will reach 100M+
   rows; sequential scan = production outage.
   **Fix:** Composite index + see #6.

6. **Audit log has no partitioning or archival job**
   Migration adds `archived_at` column but no scheduled job archives.
   **Fix:** Convert `audit_log` to `PARTITION BY RANGE (created_at)` with
   monthly partitions; cron-style job moves partitions > 90 days to cold storage.

7. **`billing_records` queries leak across workspaces** (`backend/nexus/api/billing.py:71`)
   `SELECT BillingRecord WHERE created_at >= ?` with no `workspace_id` filter.
   Both a tenant-isolation bug **and** a performance bug.
   **Fix:** Add `.where(BillingRecord.workspace_id == workspace_id)` — and
   confirm RLS coverage once §1 finding #2 is fixed.

### Medium

8. **`human_approvals(agent_id, status)` not indexed**
9. **`audit_log` count uses subquery materialization** (`audit.py:90`) — use direct `COUNT(*)` with same WHERE clause
10. **`semantic_memory` and `episodic_memory` also need ivfflat for any future vector queries**

### Notes — looks correct

- Connection pool config: `pool_pre_ping=True`, `pool_size`, `max_overflow`, `pool_recycle` ✓
- All `db.execute()` calls properly awaited ✓
- Pagination present on all list endpoints ✓
- Existing indexes on `tasks(agent_id, created_at)`, `episodic(agent_id, created_at)`, `llm_usage(agent_id, created_at)` are correct ✓
- `workspace_files.embedding` has ivfflat — pattern to copy for episodic ✓

## 3. Backend correctness — findings

### Critical

1. **Heartbeat task may be garbage-collected** (`backend/nexus/agents/base.py:120`)
   `heartbeat_task = asyncio.create_task(...)` — local var, no reference held.
   GC can collect it; agent silently stops heartbeating; health monitor
   auto-fails the agent.
   **Fix:** `self._heartbeat_task = asyncio.create_task(...)`.

2. **Memory write failure publishes result anyway** (`backend/nexus/agents/base.py:332` + outer `except` at `:411`)
   Doc invariant: "memory must succeed before publish." Code: any exception in
   `write_memory()` is caught by the outer broad `except Exception` which then
   publishes an error result and marks the task complete.
   **Fix:** Wrap memory write in its own try block; on failure, re-raise with a
   sentinel that the outer handler routes to dead-letter without publishing a
   user-visible result.

3. **Result consumer has no idempotency check** (`backend/nexus/core/kafka/result_consumer.py:46`)
   Kafka at-least-once delivery + no `check_idempotency()` = double-completion
   on redelivery.
   **Fix:** `await check_idempotency(f"result:{message_id}")` early-return on duplicate.

4. **Consumer error handler can leak active tasks** (`backend/nexus/agents/base.py:154`)
   If `_handle_consumer_error()` itself raises, the task stays in `_active_tasks`
   forever, blocking graceful shutdown.
   **Fix:** Move unregister into a `try/finally` around the entire processing
   block.

### High

5. **Recovery service re-publish not idempotent** (`backend/nexus/core/recovery.py:90`)
   Crash mid-recovery → next startup re-publishes the same orphan again.
   **Fix:** Set `recovery_attempted_at` on the task before publishing; skip
   if non-null in the current run.

6. **Health monitor silence check has a race** (`backend/nexus/agents/health_monitor.py:125`)
   Non-atomic fetch-then-compare can false-positive auto-fail a healthy agent.
   **Fix:** Redis Lua script for atomic check-and-fail.

7. **KeepSave client reads from `os.environ` directly** (`backend/nexus/integrations/keepsave/client.py:19`)
   Violates §16 rule 6.
   **Fix:** Read via `settings` only.

8. **Broad `except Exception` swallows `CancelledError`** (`backend/nexus/agents/base.py:411`)
   Graceful shutdown can't propagate.
   **Fix:** `except asyncio.CancelledError: raise` at top.

9. **Circuit breaker keeps old failures after recovery** (`backend/nexus/core/llm/circuit_breaker.py:214`)
   HALF_OPEN → CLOSED clears `_failure_count` but not the sliding window.
   Next call rapidly trips OPEN again.
   **Fix:** Clear the window on successful recovery.

### Medium

10. **Prompt reload hits DB every task** (`backend/nexus/agents/base.py:520`) — cache hash with 5-min TTL
11. **Path-traversal guard uses Python 3.9+ `is_relative_to()`** (`backend/nexus/tools/adapter.py:148`) — pin Python ≥ 3.12 in `pyproject.toml` and document it
12. **No timeout on LLM meeting responses** (`backend/nexus/agents/base.py:850`) — `asyncio.timeout(60)` wrapper

### Notes — looks correct

- Idempotency guard on agent side via Redis `SET NX` ✓
- Dead-letter routing after 3 retries ✓
- 30-second drain timeout ✓
- Broadcast best-effort pattern ✓

## 4. Tool / MCP integration — findings

### Critical

1. **Irreversible-tool approval guards are missing from the wrapped functions** (`backend/nexus/tools/adapter.py:494–757`)
   `tool_file_write`, `tool_send_email`, `tool_git_push`, `tool_hire_external_agent`
   have **zero** calls to `await require_approval(...)`. They execute side
   effects unconditionally. A code comment on line 490 claims approval is
   enforced "by the guard chain" — there is no such chain in `agents/base.py`,
   and Pydantic AI has no built-in approval mechanism.
   **This is the single most dangerous finding in this audit.** An LLM can write
   to disk, push commits, send emails, and spend money via A2A with no human
   in the loop.
   **Fix:** Add as the first executable line of each wrapped tool:
   ```python
   await require_approval(ctx, IrreversibleAction(
       action="file_write",
       description=f"Write {len(content)} chars to {path}",
       task_id=ctx.deps.task_id,
   ))
   ```

2. **Plugin tools bypass `requires_approval` flag** (`backend/nexus/integrations/plugins/registry.py:235`)
   `PluginTool.execute()` accepts the flag from the manifest but never reads it
   when deciding whether to call `require_approval()`.
   **Fix:** `if self.requires_approval: await require_approval(...)` before execute.

3. **Stale tool-registry test** (`backend/nexus/tests/unit/test_tools_registry_phase2.py:26`)
   Asserts an outdated 5-tool list for Engineer; current registry grants 14+.
   The test passes only because the registry returns a superset — assertion
   logic must be updated, otherwise drift goes undetected.

### High

4. **Tool output goes through size-cap, not PII sanitizer** (`backend/nexus/tools/adapter.py:32`)
   `_sanitize_tool_output()` truncates at 50KB; it does **not** redact PII.
   `nexus.core.sanitization.sanitize_output()` exists but is not called in any
   tool wrapper.
   **Fix:** Replace `_sanitize_tool_output()` with `sanitize_output()` in all 13
   call sites, or chain them.

5. **Runtime tool-access enforcement absent** (`backend/nexus/agents/base.py`)
   Tools are filtered at agent construction. There is no runtime check that
   intercepts a tool call from a misregistered or globally-registered tool.
   **Fix:** Wrap each tool with a decorator that asserts
   `tool.__name__ in agent.tool_access` before invocation.

6. **`tool_send_email` is a stub** (`backend/nexus/tools/adapter.py:512`)
   Logs and returns a mock confirmation. No SMTP/SendGrid integration. Agents
   believe email was sent.
   **Fix:** Implement real SMTP, or raise loudly with "email integration not configured."

### Medium

7. **E2B sandbox unconfigured → silent string-error returns** (`backend/nexus/tools/sandbox/client.py:66`)
   If `E2B_API_KEY` is missing, tools return `"E2B sandbox not configured"` as a
   string. Agent thinks the call succeeded with that text.
   **Fix:** Remove sandbox tools from the registry at startup if unconfigured.

8. **`tool_git_push` has no pre-flight repo or branch validation** (`backend/nexus/tools/adapter.py:704`)
   `git add -A` commits everything, including secrets accidentally left in the workspace.
   **Fix:** Validate repo, list staged files before commit, refuse if `.env` or `*key*` patterns match.

9. **`tool_hire_external_agent` does not validate external response** (`backend/nexus/tools/adapter.py:529`)
   External agent's `result.output` is returned to the calling NEXUS agent's
   context unchecked — prompt-injection vector.
   **Fix:** Validate against a strict response schema; truncate/sanitise.

### Notes — looks correct

- All 17 tool functions assigned to ≥ 1 role; no dead code ✓
- Most docstrings follow one-line + Args + Returns pattern ✓
- E2B sandbox `.kill()` in `finally` ✓
- CEO has `[]` tools by design ✓

## 5. Recommended fix order

Fix in this order — earlier items unblock later ones.

| # | Finding | Effort | Reason for priority |
|---|---------|--------|---------------------|
| 1 | Add `require_approval()` to all 4 irreversible tools (Tools §1) | 30 min | An LLM can currently write to disk, send emails, push commits. Until fixed, **disable these tools** by removing from registry. |
| 2 | Plugin `requires_approval` flag honored (Tools §2) | 15 min | Same risk surface |
| 3 | Hook `set_rls_context()` into Litestar session lifecycle (Security §2) | 1 h | Until fixed, multi-tenant isolation is application-layer only |
| 4 | API key endpoints use injected session (Security §5) | 15 min | Closes the most direct RLS bypass |
| 5 | Encrypt OAuth tokens at rest (Security §1) | 1 h | Backfill existing rows in a migration |
| 6 | OAuth state CSRF parameter (Security §3) | 1 h | |
| 7 | Result consumer signature validation + idempotency (Security §7 + Backend §3) | 30 min | Two bugs, same file |
| 8 | Memory-write-failure invariant (Backend §2) | 1 h | Test with a chaos test |
| 9 | Heartbeat task `self._heartbeat_task` (Backend §1) | 5 min | |
| 10 | Add 6 missing indexes (DB §1–6) | 1 h | Single alembic migration |
| 11 | Audit log partitioning + archival job (DB §6) | 4 h | Schedule before prod scale |
| 12 | Hardcoded dev signing key fallback (Security §6) | 15 min | |
| 13 | Billing queries scope to workspace (DB §7) | 30 min | |
| 14 | Tool output PII sanitizer (Tools §4) | 30 min | |
| 15 | LLM injection classifier fail-closed (Security §9) | 15 min | |
| 16 | A2A token PBKDF2 hashing (Security §4) | 1 h | Backfill |
